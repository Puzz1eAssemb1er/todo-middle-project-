# QA Middle Project: UI + API Testing

## Описание проекта

Этот репозиторий — учебно-практический проект уровня Middle QA Engineer.  
Цель проекта — продемонстрировать полный цикл тестирования веб-приложений:
- функциональное тестирование UI (ToDoMVC),
- тестирование REST API (Swagger Petstore),
- базовые проверки данных,
- автоматизация тестов (Python, pytest, requests, Selenium).

Проект оформлен так, чтобы им можно было делиться с рекрутерами и тимлидами как с примером реального рабочего подхода к тестированию.

---

## Стек и инструменты

- Язык: Python 3.x  
- Тестовый фреймворк: pytest  
- UI-автоматизация: Selenium WebDriver  
- API-автоматизация: requests  
- Инструменты для ручного тестирования:
  - Postman (API)
  - браузерные DevTools (UI)
- Контроль версий: Git + GitHub

## Объекты тестирования

### UI: ToDoMVC (React)
- Сайт: https://todomvc.com/examples/react/
- Основной функционал:
  - добавление задач,
  - редактирование,
  - удаление,
  - отметка выполнения,
  - фильтрация задач,
  - очистка выполненных задач.

### API: Swagger Petstore
- Сайт: https://petstore.swagger.io/
- Основной функционал:
  - создание сущностей (pet),
  - чтение данных,
  - обновление,
  - удаление,
  - позитивные и негативные сценарии.

## Приоритеты тестирования (UI ToDoMVC)

| Приоритет | Функциональность | Описание |
|----------|------------------|----------|
| **P1 — Критический** | Добавление задач | Основная функция приложения. Без неё ToDoMVC теряет смысл. |
| **P1 — Критический** | Выполнение / снятие выполнения | Ключевая логика работы со списком задач. |
| **P1 — Критический** | Удаление задач | Позволяет управлять списком и поддерживать его актуальность. |
| **P1 — Критический** | Отображение списка задач | Базовый UI-функционал, без которого приложение не работает. |
| **P1 — Критический** | Счётчик задач | Важная часть UX, отражает текущее состояние списка. |
| **P2 — Основной** | Редактирование задач | Важная функция, но не блокирующая работу приложения. |
| **P2 — Основной** | Фильтры (All / Active / Completed) | Улучшают UX, но не критичны для базового сценария. |
| **P2 — Основной** | Clear completed | Удобная функция, но не критичная. |
| **P3 — Негативные сценарии** | Пустые задачи | Проверка валидации. |
| **P3 — Негативные сценарии** | Задачи из пробелов | Проверка обработки некорректного ввода. |
| **P3 — Негативные сценарии** | Очень длинные строки | Проверка граничных значений. |
| **P3 — Негативные сценарии** | HTML / спецсимволы | Проверка безопасности и корректности отображения. |
| **P3 — Edge cases** | 0, 1, 100 задач | Проверка поведения на границах. |
| **P3 — Edge cases** | Все задачи выполнены / активны | Проверка корректности фильтров и счётчика. |
